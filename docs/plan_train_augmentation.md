# Baseline Dataset 구축 계획: 전처리 및 증강 (Phase 1)

> **목적**: 원본 데이터(`train.csv`)에서 검증셋을 엄격히 격리한 후, 훈련셋을 클래스당 2,500~3,000건 규모로 증강하여 견고한 베이스라인 데이터셋을 구축함.
> **참조 문서**: `docs/strategy.md` (데이터 전략), `docs/DLThon.md` (제약 사항)

---

## 1. 환경 설정 및 데이터 로드
- **라이브러리**: `pandas`, `scikit-learn` (분할), `konlpy` (Okt/Mekab), `random`, `tqdm`.
- **데이터 로드**: `data/train.csv` 읽기.

## 2. 데이터 전처리 (Deduplication)
- **완전 중복 제거**: `conversation` 기준 동일 텍스트 제거 (약 104건 예상).
- **준-중복 제거**: TF-IDF 코사인 유사도 0.95 이상인 데이터 검토 및 제거 (약 117건 예상).
- **결과**: 클래스당 약 900~950건의 고유 데이터 확보.

## 3. 검증셋(Validation Set) 격리 — 최우선 단계
- **방법**: `StratifiedShuffleSplit` 사용하여 클래스별 비율 유지.
- **할당량**: 클래스당 **약 200개(20%)**를 무작위 추출하여 격리.
- **저장**: `data/val_holdout.csv` (증강 전 순수 원본 데이터).
- **주의**: 이 데이터는 이후 어떤 증강 파이프라인에도 포함되지 않아야 함.

## 4. 데이터 증강 (Training Set Only)
- **대상**: 검증셋을 제외한 남은 훈련 데이터 (클래스당 약 700~800개).
- **목표량**: 클래스당 **2,500 ~ 3,000건** (원본 포함 약 3~4배 증강).
- **증강 기법 (Strategy)**:
    1. **동의어 교체 (SR)**: 핵심 키워드를 제외한 일반 명사/동사를 WordNet 또는 유의어 사전으로 교체.
    2. **문장 순서 셔플 (RS)**: 대화 내에서 인접한 발화(Turn)의 순서를 바꿈.
    3. **역번역 (BT)**: 한국어 → 영어 → 한국어 번역으로 자연스러운 문장 변형.
    4. **무작위 삽입 (RI)**: 일상적인 추임새(아, 음, 하하 등)를 문맥에 맞게 삽입.
- **보존 규칙**: `strategy.md`에 정의된 클래스별 핵심 키워드(예: "돈", "제발", "부장님")는 증강 과정에서 변형되지 않도록 보호.

## 5. 데이터 병합 및 최종 검토
- **병합**: 원본 훈련 데이터 + 증강 데이터.
- **클래스 밸런싱**: 모든 클래스의 총 합계가 3,000건에 수렴하도록 조정.
- **파일명**: `data/train_augmented_v1.csv` 저장.

## 6. 데이터 품질 검증 (Checklist)
- [x] 훈련셋과 검증셋 사이에 중복 문장이 존재하는가? (Data Leakage 체크) -> 통과 (겹침 0건)
- [x] 증강된 문장이 해당 클래스의 핵심 의미를 유지하고 있는가? -> 통과 (키워드 보존율 확인)
- [x] 특수문자(`!`, `?`) 분포가 원본의 특징을 따르고 있는가? -> 통과 (평균 분포 일치)
- [x] 일반 대화(합성 예정)를 위한 공간이 확보되었는가? -> 확보됨 (TARGET_COUNT 3000 통일)

### Step-by-Step 실행 로드맵 (in `baseline_dataset.ipynb`)
1. **Cell 1**: EDA 요약 수치 확인 및 환경 설정.
2. **Cell 2**: 중복 제거 및 데이터 클렌징.
3. **Cell 3**: 검증셋(20%) 분리 및 로컬 저장.
4. **Cell 4**: 증강 모듈(SR, RS, BT 유틸리티) 정의.
5. **Cell 5**: 클래스별 루프를 돌며 목표치까지 증강 수행.
6. **Cell 6**: 데이터 분포 시각화 및 최종 CSV 저장.
