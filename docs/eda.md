### 📊 NLP 프로젝트를 위한 심층 EDA 분석 지표 리스트

| 분석 카테고리 | 세부 분석 지표 | 분석 목적 및 상세 내용 | 도출 가능한 전략적 인사이트 (Action Item) |
| :--- | :--- | :--- | :--- |
| **1. 길이 및 구조 (Structure)** | **문장/단어/토큰 길이 분포** | 클래스별 평균/중간값/최대·최소 길이 확인 (히스토그램) | 특정 클래스만 너무 길거나 짧을 경우, 길이에 의한 편향(Bias) 제거 증강 필요성 판단 |
| | **대화 발화 횟수 (Turns)** | 하나의 대화(idx)당 몇 번의 발화(말 바꿈)가 일어나는지 통계 | 합성 데이터 생성 시 모델이 학습할 적절한 '대화의 호흡' 설정 |
| **2. 어휘 및 키워드 (Vocabulary)** | **TF-IDF 상위 키워드** | 빈도수를 넘어 해당 클래스를 '대표'하는 고유 단어 추출 | 합성 데이터 생성 시 반드시 포함하거나, 다른 클래스에서 배제할 단어 리스트업 |
| | **N-gram 분석 (Bi/Tri-gram)** | 두 개 이상의 단어 조합(예: '돈' + '내놔') 빈도 확인 | 단어 하나가 아닌 '문맥적 패턴' 파악. 프롬프트에 구체적 상황 묘사 시 활용 |
| | **어휘 다양성 (TTR)** | 전체 토큰 대비 고유 토큰의 비율 (Type-Token Ratio) | 특정 클래스가 너무 뻔한 단어만 쓰는지 확인. 증강 시 단어 교체의 폭 결정 |
| **3. 클래스 관계 (Semantic)** | **클래스 간 코사인 유사도** | 클래스별 대표 벡터 간의 거리 계산 | 모델이 가장 헷갈려 할 '클래스 쌍' 식별. 해당 쌍 사이의 경계선 강화 전략 수립 |
| | **중복 단어 (Overlap) 분석** | 클래스 간 교집합 단어 및 비율 확인 (**Hard Negatives**) | '돈'이 갈취와 일반 대화에 동시에 나올 때, 이를 구별할 '결정적 차이'를 합성 데이터에 반영 |
| **4. 언어적 특징 (Linguistic)** | **품사(POS) 분포** | 명사, 동사, 형용사, 감탄사의 비중 확인 | 협박은 '명령조 동사', 괴롭힘은 '부정적 형용사'가 많은지 확인하여 말투(Tone) 제어 |
| | **존칭/비속어 사용 빈도** | '합니다' vs '해', 욕설이나 비속어의 클래스별 출현 빈도 | '일반 대화' 생성 시 존댓말과 반말의 비중을 트레인 셋과 일치시켜 이질감 제거 |
| **5. 품질 및 노이즈 (Quality)** | **특수문자/문장부호 빈도** | "!", "?", "...", "ㅠㅠ" 등의 사용 패턴 | 전처리 시 특수문자를 제거할지, 아니면 모델의 힌트로 남길지 결정 |
| | **데이터 중복 및 유사도** | 트레인 셋 내에 사실상 똑같은 문장이 있는지 확인 | 과적합 방지를 위한 데이터 클렌징 및 유니크한 합성 데이터 확보 |
| | **불용어(Stopwords) 분석** | 의미 없는 기능어들이 특정 클래스에 편향되어 있는지 확인 | 모델이 '을/를' 같은 조사에 의존하지 않도록 불용어 처리 수위 조절 |
| **6. 상황 및 맥락 (Context & Flow)** | **도메인 별 키워드 쏠림 분석** | 특정 장소(회사, 학교, 길거리) 관련 단어와 클래스 간 상관관계 분석 | '회사' 단어가 나오면 무조건 '직장 내 괴롭힘'으로 분류하는 편향 방지. 일반적인 사무 대화 데이터 설계 시 활용 |
| | **발화 순서별 감정/의도 변화** | 대화 초반/중반/후반의 어조 변화 및 공수(공격/방어) 교대 패턴 확인 | 처음엔 친절하다가 마지막에 위협적으로 변하는 '간접 협박' 등 고난도 시나리오 보강 및 증강 전략 수립 |
