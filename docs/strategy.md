## 1. 데이터 전처리 전략

### 1-1. 중복 제거 (Deduplication) — 최우선

> 근거: `[5-2]` 완전 동일 중복 **104건**, 준-중복(유사도≥0.95) **117건** 발견

| 작업 | 방법 | 기대 효과 |
|---|---|---|
| 완전 중복 제거 | `df.drop_duplicates(subset=['conversation'])` | 과적합 방지, 학습 효율 향상 |
| 준-중복 제거 | TF-IDF 코사인 유사도 ≥0.95인 쌍 중 하나 제거 | 모델이 '외운' 패턴에 의존하는 것 방지 |

**중복 제거 후 예상 데이터 크기**: 약 3,800건 내외 (기존 ~4,000건에서 ~200건 감소)

---

### 1-2. 특수문자/이모티콘 처리

> 근거: `[5-1]` 모든 클래스에서 `emoticon_len = 0.0`, `ellipsis_count = 0.0`

| 특수문자 | 처리 방침 | 근거 |
|---|---|---|
| `ㅋㅋ`, `ㅠㅠ` | **학습 데이터에 없으므로 합성 시에도 사용 금지** | emoticon_len = 0.0 (전 클래스) |
| `...` (말줄임) | **합성 시 사용 금지** | ellipsis_count = 0.0 (전 클래스) |
| `!` (느낌표) | **유지** — 클래스 구분 힌트 가능 | 협박(1.85) > 갈취(1.02) > 기타 괴롭힘(0.98) > 직장 괴롭힘(0.70) |
| `?` (물음표) | **유지** | 전 클래스 4.08~4.87로 고르게 분포 |

---

### 1-3. 불용어(조사) 처리 — 선택적

> 근거: `[5-3]` CV > 0.3인 편향 조사 6개 발견

```
'과': CV = 0.697  ← 특정 클래스에 극단적 쏠림
'를': CV = 0.447
'을': CV = 0.368
'의': CV = 0.356
'하고': CV = 0.346
'잖아': CV = 0.319
```

| 전략 옵션 | 설명 |
|---|---|
| **옵션 A (권장)**: 제거하지 않음 | 조사는 문맥 정보를 포함하며, BERT 계열 모델은 이를 활용 가능. 단, Ablation study에서 비교 실험 |
| **옵션 B**: CV > 0.4인 조사만 제거 | `'과'`, `'를'`만 선택적 제거하여 편향 완화 |

## 2. 데이터 증강 전략 (위협 4클래스)

> **규칙**: 새로운 데이터 생성 불가 → 기존 데이터의 **변형(Augmentation)만** 허용

### 2-1. 증강 기법 선정

| 기법 | 방법 | 적용 근거 |
|---|---|---|
| **동의어 교체** (Synonym Replacement) | 핵심 키워드를 유지하면서 주변 단어를 동의어로 교체 | TTR이 전 클래스 0.90 수준으로 높음 → 소폭 교체로도 새로운 패턴 생성 가능 |
| **문장 순서 셔플** (Sentence Shuffle) | 대화 내 발화 순서를 소폭 변경 (인접 2개 발화 swap) | 발화 횟수 median=10으로 충분한 셔플 공간 존재 |
| **역번역** (Back Translation) | 한→영→한 번역으로 표현 다양화 | 동의어 교체보다 자연스러운 변형 가능 |
| **랜덤 삽입/삭제** | 의미 없는 추임새 삽입 또는 부정적 의미 없는 단어 삭제 | 길이 분포(char_len 41~874)가 넓어 다양한 길이 시도 가능 |

### 2-2. 증강 시 반드시 보존할 핵심 키워드

> 근거: `[2-1]` TF-IDF 상위 키워드 + `[2-2]` N-gram 분석

| 클래스 | 보존 필수 키워드 (증강 시 변경 금지) | 보존 필수 N-gram 패턴 |
|---|---|---|
| **협박** | 제발, 니가, 지금 | `지금 당장`, `제발 살려주세요`, `시키는 대로`, `죽고 싶지 않으면` |
| **갈취** | 돈이, 안돼, 없어 | `뒤져서 나오면`, `돈이 없어`, `맞고 줄래`, `줄래 그냥 줄래` |
| **직장 내 괴롭힘** | 부장님, 죄송합니다, 아닙니다, 제가 | `정말 죄송합니다`, `이건 아닌 같습니다`, `열심히 하겠습니다` |
| **기타 괴롭힘** | 아니야, 그렇게 | `말이 너무`, `무슨 소리야`, `제발 그만해` |

### 2-3. 증강량 설정

> 근거: `[5-2]` 중복 제거 후 약 3,800건 → 클래스당 ~950건 예상

| 클래스 | 중복제거 후 예상 | 증강 목표 | 증강 배율 |
|---|---|---|---|
| 협박 | ~950 | 1,200 | ×1.3 |
| 갈취 | ~950 | 1,200 | ×1.3 |
| 직장 내 괴롭힘 | ~950 | 1,200 | ×1.3 |
| 기타 괴롭힘 | ~950 | 1,200 | ×1.3 |

> 과도한 증강은 오히려 과적합을 유발할 수 있음. ×1.3~×1.5 수준이 안전.

## 3. 합성 데이터 생성 전략 (일반 대화)

### 3-1. 합성 데이터의 물리적 스펙

> 근거: `[1-1]`, `[1-2]` 길이 및 발화 통계

| 파라미터 | 설정값 | 근거 |
|---|---|---|
| **생성량** | **1,000~1,200건** | 위협 클래스당 ~1,000건과 균형 |
| **글자 수** | 180~250자 (median 기준) | 4클래스 median 범위: 185~220 |
| **단어 수** | 45~60개 | 4클래스 median 범위: 47~55 |
| **발화 횟수** | **10턴** (±2) | 4클래스 모두 median = 10 |
| **느낌표 사용** | 대화당 0~2개 | 위협 클래스 평균: 0.70~1.85 |
| **물음표 사용** | 대화당 3~5개 | 위협 클래스 평균: 4.08~4.87 |
| **이모티콘/말줄임** | **사용 금지** | 학습 데이터 전체에서 0건 |

---

### 3-2. 합성 대화의 말투 설계

> 근거: `[4-2]` 존댓말/반말/비속어 통계

```
              polite  casual  profanity
갈취 대화         1.69    6.85      0.18
기타 괴롭힘 대화     1.39    7.94      0.23
직장 내 괴롭힘 대화   3.22    6.62      0.09
협박 대화         2.07    7.68      0.51
```

**일반 대화 말투 설정**:

| 파라미터 | 설정값 | 근거 |
|---|---|---|
| 존댓말 비중 | **2.0~3.0** (polite_count) | 위협 클래스 평균 1.39~3.22의 중간값 |
| 반말 비중 | **6.5~8.0** (casual_count) | 위협 클래스 평균 6.62~7.94 범위 |
| 비속어 | **0 (절대 금지)** | 일반 대화에 욕설이 있으면 위협으로 오분류됨 |

---

### 3-3. 합성 대화의 도메인 설계 — 핵심 전략

> 근거: `[6-1]` 도메인 키워드 쏠림 분석

```
              company  school  street
직장 내 괴롭힘 대화  1.179    0.100   0.042  ← 회사 키워드 극단적 쏠림
기타 괴롭힘 대화    0.019    0.239   0.051  ← 학교 키워드 쏠림
```

**모델의 도메인 편향을 깨기 위해** 일반 대화 1,000건을 아래와 같이 도메인별로 분배:

| 도메인 | 생성 비율 | 생성 수량 | 포함할 키워드 예시 | 목적 |
|---|---|---|---|---|
| **직장 일상** | 30% | 300건 | 대리, 팀장, 회의, 출근, 업무 | `직장 = 괴롭힘` 편향 제거 |
| **학교 일상** | 20% | 200건 | 학교, 수업, 숙제, 시험, 학원 | `학교 = 괴롭힘` 편향 제거 |
| **일상/거리** | 20% | 200건 | 카페, 편의점, 지하철, 공원 | 다양한 배경 확보 |
| **자유 주제** | 30% | 300건 | 음식, 날씨, 취미, 영화, 여행 | 위협 키워드 0인 순수 대화 |

### 3-4. Hard Negative 합성 — 가장 중요한 전략

> 근거: `[3-1]` 코사인 유사도 + `[3-2]` 공통 단어 분석

**혼동 위험 순위**:
```
1위: 협박 ↔ 기타 괴롭힘 = 0.8689  ← 공통 단어 33개
2위: 협박 ↔ 갈취        = 0.8262  ← 공통 단어 30개
3위: 기타 괴롭힘 ↔ 갈취   = 0.8261  ← 공통 단어 33개
```

위협 클래스 간 **공통 빈출 단어**가 일반 대화에 등장해도 모델이 위협으로 오분류하지 않도록,
**위협 키워드가 포함되지만 무해한 문맥**의 합성 대화를 의도적으로 생성합니다:

| 위협 키워드 | 위협 문맥 (학습 데이터) | 일반 대화에 넣을 Safe 문맥 |
|---|---|---|
| `돈이` | "돈이 없어? 뒤져서 나오면 돼" | "돈이 좀 모이면 여행 가고 싶다" |
| `제발` | "제발 살려주세요" | "제발 이번 시험은 잘 봤으면" |
| `죽여` / `죽어` | "죽여버릴 거야" | (사용 금지 — 일반 대화에 부적절) |
| `지금 당장` | "지금 당장 가져와" | "지금 당장 치킨 시키자" |
| `시키는 대로` | "시키는 대로 해" | (사용 금지 — 명령조는 일반 대화에 부적절) |
| `죄송합니다` | "죄송합니다 부장님" | "죄송합니다 늦어서" |
| `맞을래` / `맞고` | "맞을래? 돈 줄래?" | (사용 금지) |
| `무슨` | "무슨 소리야" | "무슨 영화 볼까?" |
| `진짜` | "진짜 죽여버린다" | "진짜 맛있다 이거" |

**생성 규칙**:
- ✅ 사용 가능: `돈이`, `제발`, `지금 당장`, `죄송합니다`, `무슨`, `진짜` → 무해한 문맥에 배치
- ❌ 사용 금지: `죽여`, `죽어`, `맞을래`, `시키는 대로`, `내놔`, `닥쳐` → 일반 대화에 절대 포함하지 않음

### 3-5. 품사 비율 가이드라인

> 근거: `[4-1]` 품사(POS) 분포

```
          Noun    Verb  Adjective  Exclamation
갈취      0.3428  0.1548   0.0815      0.0139
기타괴롭힘  0.3471  0.1438   0.0865      0.0192
직장괴롭힘  0.3605  0.1354   0.0854      0.0105
협박      0.3537  0.1702   0.0737      0.0069

**일반 대화 품사 목표**:

| 품사 | 목표 비율 | 설계 방향 |
|---|---|---|
| 명사 | 0.35~0.37 | 위협 클래스 범위(0.34~0.36) 내에서 자연스럽게 |
| **동사** | **0.12~0.14** | 위협 클래스(0.13~0.17)보다 **약간 낮게** → 명령조 동사를 배제하므로 자연히 감소 |
| 형용사 | 0.08~0.09 | 위협 클래스와 유사하게 |
| 감탄사 | 0.01~0.02 | 위협 클래스와 유사하게 |

### 3-6. 대화 흐름(Flow) 설계

> 근거: `[6-2]` 발화 순서별 공격/방어 어조 변화

```
위협 클래스 특징: 모든 클래스에서 후반부로 갈수록 위협·방어 단어 밀도가 증가
  갈취:   0.486 → 0.525 → 0.746 (위협 후반 집중)
  협박:   0.262 → 0.325 → 0.638 (위협 급증 패턴)
```

**일반 대화 흐름 설계**:

| 구간 | 위협 단어 | 방어 단어 | 설계 방향 |
|---|---|---|---|
| 초반 | 0 | 0 | 인사, 안부, 화제 전환 |
| 중반 | 0 | 0 | 본론(일상 이야기, 의견 교환) |
| 후반 | 0 | 0 | 마무리, 약속, 인사 |

> **핵심**: 일반 대화는 **초~후반까지 위협/방어 단어가 0**이어야 합니다.
> 이것이 위협 클래스(후반부 위협 급증)와의 **결정적 차별점**입니다.

## 4. 프롬프트 설계 가이드 (합성 데이터 생성용)

위의 모든 전략을 종합하여, LLM에 일반 대화를 생성 요청할 때 사용할 프롬프트의 핵심 조건:

```
[프롬프트 필수 조건 체크리스트]

□ 대화 길이: 180~250자, 8~12턴
□ 말투: 존댓말(2~3회) + 반말(6~8회) 혼용, 비속어 0
□ 도메인: {직장 일상 / 학교 일상 / 거리·카페 / 자유 주제} 중 하나
□ 특수문자: 느낌표(0~2), 물음표(3~5), 이모티콘·말줄임 사용 금지
□ Hard Negative: '돈', '제발', '지금 당장' 등을 무해한 문맥에 자연스럽게 배치
□ 금지 단어: 죽여, 죽어, 맞을래, 내놔, 닥쳐, 시키는 대로, 패버릴 등 위협 전용 표현
□ 대화 흐름: 전 구간에서 위협/방어 단어 밀도 = 0
□ 출력 형식: 줄바꿈(\n)으로 발화를 구분하는 단일 텍스트
```

## 5. Ablation Study 실험 설계

| 실험 번호 | 변수 | 비교 조건 | 측정 지표 |
|---|---|---|---|
| **Exp-1** | 중복 제거 | (A) 중복 있음 vs (B) 중복 제거 | F1 Score |
| **Exp-2** | 증강 배율 | (A) ×1.0 vs (B) ×1.3 vs (C) ×1.5 | F1 Score |
| **Exp-3** | 합성 대화 수량 | (A) 500건 vs (B) 1,000건 vs (C) 1,500건 | F1 Score |
| **Exp-4** | Hard Negative 포함 | (A) 순수 일상만 vs (B) Hard Negative 30% 포함 | F1 Score, 혼동 행렬 |
| **Exp-5** | 도메인 분배 | (A) 자유주제만 vs (B) 도메인별 분배 | F1 Score (직장 괴롭힘 recall) |
| **Exp-6** | 불용어 제거 | (A) 전부 유지 vs (B) CV>0.4 조사 제거 | F1 Score |

## 6. 실행 우선순위

```
Phase 1 (즉시): 전처리
  └── 중복 제거 (104건 완전 중복 + 117건 준-중복)

Phase 2 (Day 1): 합성 데이터 생성
  └── 일반 대화 1,000건 생성 (도메인 분배 + Hard Negative 포함)
  └── 프롬프트 체크리스트 기반으로 LLM 활용
